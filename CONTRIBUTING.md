# Contribution Guidelines

Thank you for your interest in contributing to the UMR Insurance project. We value community contributions and welcome improvements that enhance the quality, functionality, and user experience of this application.

## How to Contribute

1. Fork the repository.
2. Clone your fork to your local machine.
3. Create a new branch for your changes.
4. Make your updates and thoroughly test them.
5. Commit your changes with descriptive commit messages.
6. Push your branch to GitHub.
7. Open a Pull Request for review.

## Reporting Issues

If you encounter a bug or issue:

- Search existing issues before creating a new one.
- Provide a clear and detailed description.
- Include steps to reproduce the issue.
- Mention your operating system and browser version.
- Add screenshots if applicable.

## Suggesting Features

We welcome feature suggestions that improve the project.

When suggesting a feature:

- Clearly explain the feature.
- Describe the problem it solves.
- Explain the expected benefits.
- Include examples whenever possible.

## Code of Conduct

All contributors are expected to:

- Treat others with respect.
- Communicate professionally.
- Welcome constructive feedback.
- Promote an inclusive environment.
- Avoid offensive or inappropriate behavior.

## Review Process

Every pull request will be reviewed to ensure:

- Code quality standards are maintained.
- Changes function correctly.
- Documentation is updated if necessary.
- The project remains consistent and maintainable.

The review team may request revisions before approving contributions.
